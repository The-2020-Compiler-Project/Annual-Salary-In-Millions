## 规范

> 规范第一版（待确定），暂定7.2日起实行。

1. 代码规范
   + 在较难理解的地方写明注释
   + 使用`utf-8`编码
   + 缩进使用`tab`，长度为4
   + 换行符使用`crlf`
   + 文件结束插入一行空行，移除行尾空格
   + 使用`vscode C/C++`内置的`WebKit`风格进行代码格式化
2. 仓库规范
   + 在`develop`分支进行开发，每次大更新`merge`到`master`分支
   + `commit message`尽量做到简单明了，统一使用中文/英文
